## Repository Architecture

**Zero runtime dependencies.** Pure static HTML/CSS/JS — no build step, no `npm install` required to use. Node 20+ is only for local dev servers and validators.

- `styles.css` — all tokens, components, dark mode, animations (primary deliverable)
- `scripts.js` — icons (ICONS array), token table, theme toggle, scroll reveal, copy
- `tokens.json` — structured token data (JSON, machine-readable)
- `VERSION` — single source of truth for version number
- `handbook.html` — live component preview (link to #components section)
- `icons.svg` — generated by `tools/generate_icons.py`, not committed manually

---

## Critical Rules (CI-Enforced)

These will cause PRs to fail:

- **Colors MUST use OKLch** — no hex, no rgb, no hsl. Use `--ds-*` tokens.
- **No hardcoded magic numbers** — use `--ds-space-*` tokens for spacing
- **No inline `style=`** — except dynamic calculations (e.g., `--d` for stagger delay)
- **No bare `oklch()` in component rules** — use `var(--ds-*)` to reference tokens. Bare oklch/hex/rgb is only allowed in `:root` token declarations and `@keyframes`.
- **CSS before HTML** — define the `ds-*` class in `styles.css` *before* referencing it in `handbook.html`, or `validate-cssref` will block your PR.
- **Every `:root` color token needs a `[data-theme="dark"]` override** — `validate-darkmode` warns but does not block. Missing overrides break dark mode silently.
- **Three version sources must match** — `VERSION` file, `tokens.json` `"version"`, and `package.json` `"version"`. `stamp_version.py` only syncs HTML/MD — update the JSON files manually.
- **Icons** — after adding to `scripts.js` `ICONS` array, run `make icons` to generate `icons.svg`, then `make icons-check` to verify sync. Commit `icons.svg`.
- **tokens.json naming** — uses dot-notation paths (`colors.neutral.bg`), while CSS uses flattened names (`--ds-color-bg`). Map manually; the validator only warns on mismatch.
- **BEM dangling modifier** — `.ds-card a--active` is invalid. Use `.ds-card-link--active` instead.
- **No empty `catch {}` blocks**
- **No duplicate HTML `id` attributes**
- **Dark mode:** never use pure `#000` — use `oklch(15% ...)` warm gray; never use light-mode accent values directly — brighten 5-10%

## Version Management

**`VERSION` file is the only source of truth.** After bumping VERSION:

```bash
make stamp-version          # sync to all HTML/MD files
make validate-versions      # verify sync
```

Or with Python directly:

```bash
python3 tools/stamp_version.py
python3 tools/stamp_version.py --check   # CI/pre-commit check
python3 tools/stamp_version.py --diff    # preview changes
```

**Cache busting:** All HTML files use `?v=X.Y.Z` query strings on `styles.css` and `scripts.js`. After changing either file, bump the version in all HTML files. `validate_versions.py` enforces this.

> **Note:** `resume.html` and `report.html` are excluded from stamp targets — they are offline templates. If you add `?v=` cache busting to them, update `tools/stamp_version.py`'s `HTML_TARGETS` list.

## Validation Commands

Run before every PR:

```bash
make validate              # all 10 validators; exit 1 blocks, exit 0/2 pass
```

Individual validators:

```bash
make validate-tokens       # tokens.json ↔ styles.css consistency
make validate-naming       # BEM / token naming conventions
make validate-html         # HTML structure and required attributes
make validate-a11y         # accessibility (img alt, heading hierarchy, contrast)
make validate-versions      # ?v= sync with VERSION file
make validate-links        # internal anchors and cross-page references
make validate-cssref       # HTML classes defined in styles.css
make validate-darkmode     # dark mode token completeness
make validate-verext       # tokens.json / package.json version sync
make validate-hardcode     # hardcoded color values (should use --ds-* tokens)
```

Exit codes: 0 = pass, 1 = blocking error, 2 = warnings only.

## Naming Conventions

| Type | Pattern | Example |
|------|---------|---------|
| CSS variable | `--ds-{category}-{name}` | `--ds-color-olive-400` |

## Adding New Components or Tokens

1. Add OKLch colors to `styles.css` `:root` AND `[data-theme="dark"]` (CSS before HTML)
2. Add corresponding entry to `tokens.json` (uses dot-notation paths, not flattened CSS names)
3. If adding icons, append to `scripts.js` `ICONS` array → run `make icons` → run `make icons-check`
4. Add preview to `handbook.html` #components section
5. Run `make validate` — all must pass (exit 1 blocks)
6. Update `tokens.json` `"version"` and `package.json` `"version"` to match VERSION
7. Run `make stamp-version` to sync `?v=` to all HTML/MD files

## Code Style

- **Code blocks:** Use `.ds-code` container with `<pre><code class="language-*">`. Prism.js handles highlighting — do NOT manually add `<span class="token-*">`.
- **Code colors:** Use `--ds-token-*` variables (defined in `styles.css`), not hardcoded colors.
- **HTML in code blocks:** Must escape as `&lt;` and `&amp;`.

## Release Process

1. Update `VERSION` file: `echo "1.6.0" > VERSION`
2. Also update `tokens.json` `"version"` and `package.json` `"version"` to match
3. Run `make stamp-version` to sync `?v=` to all HTML/MD files
4. Run `make validate`
5. Commit: `git add -A && git commit -m "chore(release): bump to v1.6.0"`
6. Tag: `git tag v1.6.0 && git push origin v1.6.0`

Tag push triggers `release.yml` — builds PDFs and creates GitHub Release automatically.
